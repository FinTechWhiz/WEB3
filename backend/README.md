# NepseLens API (backend)

FastAPI + SQLAlchemy + PostgreSQL backend for the NEPSE analytics platform.

## Quickstart (Docker, recommended)

```bash
cp backend/.env.example backend/.env
# edit backend/.env — at minimum set a real JWT_SECRET_KEY:
#   openssl rand -hex 32

docker compose up --build
```

This starts Postgres, Redis, and the API on `http://localhost:8000`.
Then, in a second terminal, run migrations and seed real data:

```bash
docker compose exec api alembic upgrade head
docker compose exec api python -m scripts.seed_sectors
docker compose exec api python -m scripts.seed_stocks
```

Visit `http://localhost:8000/docs` for interactive API docs.

## Quickstart (local, no Docker)

Requires Python 3.12+ and a running PostgreSQL instance.

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # edit DATABASE_URL and JWT_SECRET_KEY

alembic upgrade head
python -m scripts.seed_sectors
python -m scripts.seed_stocks
uvicorn app.main:app --reload
```

## Testing

```bash
pytest -v --cov=app
```

Tests run against an in-memory SQLite database (see `tests/conftest.py`),
so the suite needs no external services and is safe to run in CI. This
sandbox could not execute `pip install` (no network access), so the test
suite has been written and syntax-verified but not run end-to-end —
run it as your first step after installing dependencies to confirm.

## Adding a new migration

```bash
alembic revision --autogenerate -m "describe the change"
alembic upgrade head
```

Always read the autogenerated file before applying it — autogenerate
misses some changes (column type narrowing, some constraint renames).

## Known limitations (see root PROJECT.md for full context)

- `stocks.sector_id` is null for every row until a verified symbol→sector
  mapping is sourced — the sector taxonomy itself is seeded, but no stock
  is auto-assigned to a sector.
- `price_history` is an empty table — no OHLCV backfill until a live/EOD
  data source is connected.
- `GET /api/v1/market/overview` intentionally has no index value or
  change% field for the same reason.
